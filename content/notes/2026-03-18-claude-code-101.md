### claude code的三种模式
- shift + tab 切换模式
- 默认模式：输入框下面显示 `? + shortcuts` ， claude code 修改文件会前询问用户
- 自动模型：输入框下面显示 `accept edit on` ，allow all edit in this session
- 规划模式: 输入框下面显示 `plan mode on` , 只讨论不改文件

> 还有一个隐藏模式, 用 `claude --dangerously-skip-permissions` 进入 claude code吗，这个模型下的claude code可以随便调用终端命令！
----
### 操作bash
在claude 输入框输入 
```
! + {bash command}
```

### 使用 Plan Mode
- 用来讨论方案，确定细节，比如claude code输入
  - > 将当前的待办应用重构为使用React+Typescript+Vite 的项目 <alt + enter 换行> ，保留现有功能
  

### 后台任务
claude 可以帮把我们启动服务（开发server时）`Ctrl + B` 将服务放置到后台。后续用 `\tasks`
可以查看后台任务
 

### 版本回滚
`/rewind` , 这个命令只能回滚claude它自己写入的文件。用终端命令写入的文件没法回滚。(还是用git吧)

### 图片处理
假如你在Figma 自己花了个界面，想让claude按那个界面画，怎么办？
让直接把图片路径告诉claude code或者直接粘贴图片就好。



### 恢复历史
```
claude -c # resume and continue
claude -r # select a session to continue
```


### 使用 MCP
以 Figma 提供的MCP的使用为例:
```
# install mcp in bash
claude mcp addhttp figma https://mcp.figma.com/mcp--transport
```
然后进入 claude code, 运行
```
/mcp
```
然后选工具就好，或者你直接和claude code对话，让LLM去主动调用合适的MCP工具


### 压缩上下文
`/compact + prompt` , 比如 `/compact 重点保留规划内容`
压缩后使用 `ctrl + o` 查看压缩后的结果

### 清空上下文
`/clear` 直接删掉上下文，如果后续任务和前面没关联就可以用


### 项目记忆文件
项目写到一半，想让对话记录持久化，可以使用 `/init` 创建一个项目记忆文件 `CLAUDE.md` 。有了这个文件，之后在这个项目目录里，无论开多少session，都会有这个记忆相关的上下文。

之后可以使用命令`/memory`来继续编辑这个文件。从这个可以看到claude的memory还区分用户级和项目级的。
```
│ Select memory to edit:                                                                                                                                              
│  ❯ 1. User memory     Saved in ~/.claude/CLAUDE.md                                                                                                              
│    2. Project memory  Saved in ./CLAUDE.md
```

### Hook
这个功能运行用户在运行某个**工具**的前后，运行**自己的逻辑**(比如代码格式化).
启动命令, 按照指示配置即可
```
/hooks
```
hook可以保存为对 本机本项目/对项目/全局 生效


### Agent Skills
skills 即给大模型看的说明书，动态加载的`prompt`。
#### 创建skill
比如创建一个叫 `daily report` 的skill。
```
mkdir -p ~/.claude/skills/daily-report
vim .claude/skills/daily-report/SKILL.md
```
然后创建内容
```
-----
name: daily-report
description:生成每日开发进度日报。当用户想要总结今天的工作、写日报或汇报进度时使用此技能。
------

# daily-report
## 格式要求
请严格按照以下 Markdown 格式输出:

```Markdown
#开发日报(YYYY-MM-DD)
##今日摘要
(用一句话总结今天的主要产出)
##详细变更
### 新增功能
[模块名]详情...
### 问题修复
[模块名]详情...
```Markdown
```


回到claude code，它就会发现这个skill， 然后让它**写一个份每日总结**，就会请求使用这个skill。当然你也可以用
`/{skill-name} + {prompt}` 的方式主动发起skill


### SubAgent
一个给自己创建小弟的功能
- 创建sub agent: 
 - `/agents` -> `Create new agent` -> `Project/Personal` -> `Generate with Claude/Manual Configuration`
 - 然后写agent功能描述（按格式手写或者让claude生成） -> 选择工具权限，

创建完agent后重启请求，之后大模型会根据对话自动调用agent去执行任务

### Plugin
一个全家桶安装包，将一系列 skill, agent , hook 打包在一起。
安装插件:
```
/plugin -> '...'
```
然后就会发现自己的 `/Skills` 里包含了属于新安装的plugin的skill
